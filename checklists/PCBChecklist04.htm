  1. Process

    1. Is there an established process for requirements analysis and management?
      1. To get real requirements
      2. To manage and track changing requirements
      3. For verification and validation of system requirements against user/stakeholder requirements
      4. For software (e.g. agile methodologies)
    2. Teamwork & collaboration
      1. Is there an agreed upon workflow
      2. Is the workflow documented and the ducumentation up to date
      3. Are interfaces and upcoming issues discussed early
      4. Responsibilities are clearly defined
      5. Issues are discussed in a civil manner. (Putting pressure on people makes them succeptible to make more errors, not less.)
      6. Is there an atmosphere of trust and understanding, so that issues are discussed early and often? (The goal is to find solutions and develop better practices, not to dispense blame.)
    3. Is there a method of tracking versions/revisions?
      1. For system
      2. For schematics
      3. For PCBs
      4. For software
    4. Is source version control used? (Git, SVN etc.)
    5. Is offsite backup used?
    6. Are design decisions documented?
      1. Within or separate from the schematics/drawings/code?

  2. System

    1. What should have ESD protection?
    2. Are amplifier circuits stable?
    3. Is there a simulation/simulator of the system or subsystems?
    4. Has the design or subsystems of it been simulated?
    5. What kind of interfaces and connections are necessary?
      1. For humans?
      2. For computers / control software?
      3. For power?
      4. For other subsystems?
    6. Are there status LEDs
      1. Power
      2. Programmable
    7. Subsystems
      1. Are multiple boards needed?
      2. How are subsystems connected?
      3. Is there a breakout board for subsystem connectors?
      4. Have harnesses/wiring/connectors been identified?
        1. Have pin-outs been defined and specified?
        2. What is the max current draw?
      5. Should any subsystems be default off? #power
    8. Communication
      1. Which busses/protocols will be used?
        1. Between subsystems?
        2. Between chips/modules?
      2. Has addresses been allocated?
    9. Watchdog and reset
      1. Can the watchdog be bypassed? #testing
      2. Can it be overruled? #testing
      3. Can it be emulated? #testing
      4. What interval is necessary? #safety
      5. Is the worst case behavior documented?
      6. Should reset be pulled to default on or off?
    10. System characteristics
      1. Is there a power budget?
      2. Estimated power dissipation of each subsystem?
      3. Estimated current draw of each subsystem?
      4. Estimated expected operating temperature?
    11. Power
      1. How is power provided?
      2. Is power-up well-defined? Some subsystems might need to be unpowered before other subsystems are powered, e.g. LED drivers unpowered until the software has booted.
        Certain components such as complex microprocessors, FPGAs and other complex ICs need special power sequencing to operate correctly; consult datasheets for details.
      3. Is power-down well-defined?
      4. Should any subsystems be isolated?
      5. Which voltage levels are needed?
      6. Which current draws are needed?
      7. Is there a shared power supply?
        1. Can it deliver sufficient power for all connected subsystems?
      8. Is the ground connection made first in hot-plugging connectors?
      9. Is a separate earth necessary?
      10. Will under or over voltage protection be needed?
    12. Clocks and oscillators
      1. Frequency linewidth/jitter are within tolerances across all environments? Clocks and oscillators are not perfect. Their frequency jitter can vary with temperature and other environmental factors.
      2. Driving ICs support crystals if crystals are used?
      3. Input rules for clocks are followed? FPGAs' have multi clock capable input pins, but the features available differ from pin to pin. Source: Choosing a Clock Input Pin for an FPGA
    13. Testing
      1. Test points on PCB for critical circuits?
      2. Test pads for flying probe/bed-of-nails/pogos setups?
      3. Has a test procedure been written?
      4. Are special connectors needed for testing?
      5. Are special peripheral circuitry needed for testing?
        1. Breakout boards for connectors?
      6. Is special equipment needed for testing?
      7. Should there be in-circuit testing?
        1. Voltage monitoring?
        2. Current monitoring?
        3. Temperature monitoring?
        4. Peripheral device behavior
        5. Board revision or version The board revision or version can be made software readable using either GPIO or shift register pins tied to ground or supply. An ADC can also be used to read a voltage spanider. This can also be used for board specific functionality. 5 Tips for Versioning Embedded Systems - Design News
      8. Is there event/data logging?
        1. Voltage monitoring?
        2. Current monitoring?
        3. Temperature monitoring?
        4. Connections/disconnects?
        5. Peripheral device states?
        6. Battery health?
    14. Maintainability
      1. Is disassembly easy?
      2. Is reassembly easy?
    15. Safety
      1. Fuses
        1. Appropriately sized?
        2. Fast enough for the load?
        3. Replaceable when board is assembled
        4. Replaceable when devices is assembled
        5. Storage for spare fuse?
      2. Can connections be mated in an unsafe way?
      3. Do connectors for different purposes share the same type of plug/socket?
      4. Is signal and power avoided in same connector? Wide Band Laboratory Fire, Fermi National Accelerator Laboratory, 1987.
        A misjoined connector resulted in several amps being drawn through cables rated for only 1 A. A fire started from this. Source: Wide Band Laboratory Fire
      5. safety regulations
        1. shock hazard warnings and prevention / covers
        2. radiated energy warnings and shields
        3. applicable standards checked
        4. protection against liquids and foreign objects
        5. removed / opened casing detected and handled
        6. laws and industry standards observed
    16. EMC
      1. Requirements
        1. What noise frequencies are emitted? (e.g. clock and SMPS frequencies)
        2. What noise frequencies is the circuit sensitive to?
        3. What noise will be generated by connected devices? (e.g. mains adaptors)
        4. What countries will the product be sold in? What standards cover these countries?
        5. What is the end use for the product? Does this have any specific EMC requirements ? Industrial or automotive use has different test requirements and permissable emission levels than consumer goods. For industrial, the levels are usually higher. See Conformance UK's guide
      2. Prevention
        1. Do switching supplies have (or have an option for) RC snubbers? While not always needed for stability, an RC snubber can dampen ringing oscillations, which result in high frequency noise that may exceed EMC limits.
        2. Is differential signalling used for digital signals where possible? Differential signals are not only more immune to noise, but emit less as well. For high speed, long signal lines (e.g. remote displays in automotive), the difference can be considerable.
        3. Is differential signalling used for analog signals?
        4. Are shielded connectors needed?
        5. Is a shielded case needed? How will this be bonded to ground?
        6. Are switched-mode power supplies synchronized? If switch mode power supplies are powered from the same source and are not syncronized, they can produce "beat" frequencies from being out of phase, and input and output ripple is also increased. See TI Power Tip 73: Synchronizing Makes for Well-Behaved Power Supplies
    17. Failure mode analysis
      1. Loss of ground pins on connector
      2. Effect of lost connection
        1. Between subsystems
        2. To computer
      3. System behavior when battery is fully discharged?
      4. Effects of voltage transients and high voltages on FETs
      5. Expected failure modes of failed semiconductors
        1. Expected effects of failed semiconductors
      6. Are component ratings derated by expected operating temperature/voltage/current?
      7. Environmental tolerance
        1. Vibration
        2. Heating
        3. Radiation
        4. Humidity
        5. Magnetism

  3. Components

    1. Are the necessary components in stock?
      1. With a margin for defects/failures/loss?
      2. With a margin for spill from Pick-and-Place machine?
    2. Are voltage ratings of components sufficient?
    3. Part obsolescence review
    4. Updated pricing info checked
    5. ROHS compliance requirement review
    6. Are there multiple sources?
    7. Are there alternate manufacturers?
    8. Are suitable alternatives identified?
    9. Have errata sheets been checked?
    10. Is reset active-high or active-low?
    11. Do any pins need biasing / pull-up / pull-down / strap?
      1. During initialization?
      2. For addresses?
    12. Are some functions only available in certain modes?
    13. Are the inputs and outputs organized in banks?
    14. Check pin numbers of all custom-generated parts
    15. Component/Part/Symbol/Footprint review passed

  4. Schematic

    1. Set up
      1. Has DRC been set up and configured?
      2. Has a grid size been picked
      3. Has paper sizes been selected for sheets?
    2. Symbols
      1. Are explicit and informative
      2. Do they resemble electrical circuit symbols?
      3. Mark internal pull-up/-down
      4. Mark internal termination
      5. Reflect the functionality or logical structure of the component In contrast to placing pins as the physically appear on the device. Schematics should reflect functionality; board layout should reflect physical reality. Physical pin layout has its merit in making debugging easier -- however, so does having a clearer schematic.
      6. Pins are assigned the correct type (passive, power, in, out etc.)
      7. Active-high and active-low marked consistently
      8. Inverting inputs marked consistently
      9. Power (and ground) pins consistently placed and marked
      10. Do pin positions adhere to the selected grid size If the the pins are off grid, connections might not be made.
    3. Functionality
      1. Are all pins on all ICs handled?
        1. Beak-out of extra pins from ICs or subsystems?
      2. Have necessary inputs been ESD protected? #ESD A typical solution would be current limiting resistors, clamping diodes.
      3. Multipart components are identified and utilized
      4. race conditions checked
      5. Diagnostic resources by design (leds, serial ports, etc.) even if unpopulated by default
    4. Electrical Components
      1. Check Absolute Maximum Ratings of every component, even if there's no reason to believe the ratings have been violated. Also look at specified (dm, cm) input levels.
      2. check the data sheet fine print and apnotes for weird IC behaviors
      3. Voltage ratings of all components checked (including derating and spike values)?
      4. automotive powered devices must withstand 60 to 100 volt surges
      5. Each IC has a predictable or controlled power-up state
      6. Polarized components checked for polarity
      7. Polarized components protected/ensured against from reverse voltage
      8. Pull-ups on all open-collector outputs (IC and discrete)
      9. check maximum power dissipation at worst-case operating temperatures
      10. ensure resistors/capacitors are operating within their specified (voltage, current), power range plus safety factor
      11. resistor power ratings derated for elevated ambient temperatures
      12. Avoid low-impedance sources driving tantalum capacitors, leading to premature failure
        Switch-on current should also be limited. References:
        • A Study of Field Crystallization in Tantalum Capacitors and its effect on DCL and Reliability - AVX
        • Voltage Derating Rules for Solid Tantalum and Niobium Capacitors - AVX
        • What a cap-astrophe! - Jim Keith, EDN
        • DC LEAKAGE FAILURE MODE - Vishay
      13. electrolytic/tantalum capacitor temperature/voltage derating sufficient for MTBF
      14. Pitfalls & gotchas
        1. piezo elements generate voltages (when shocked) that can destroy their drivers — check for susceptibility
      15. Schematic tool quirks
        1. Check hidden power and ground connections (tool/usage dependent)
        2. wires exist between all connected pins/ports (no direct pin/pin connections) if capture package does not like such connections
        3. ensure schematic software did / did not separate Vcc from Vdd, Vss from GND as needed
    5. Connectors
      1. Mating connectors on different boards match in pin-out (consider whether 1:1, 1:2, 1:(N-1), 1:N cable wiring is used and document it)
      2. Determine effect of losing each of multiple grounds on a connector
      3. Ground made first and breaks last for hot pluggability
    6. Transistor circuits
      1. avoid reverse base-emitter current/voltage on bipolar transistors (even short term, permanently damages transistor beta)
      2. check for voltage transients and high voltages on FET gates
      3. Check drain-source orientation of FETs, check channel type (ambiguous symbol variants for IC and discrete transistors!), consider body diode
    7. Power supply
      1. sufficient power rails for analog circuits
      2. check for input voltages applied with power off and CMOS latchup possibilities
      3. sufficient capacitance on low dropout voltage regulators
      4. estimate total worst case power supply current
      5. Is there sufficient bulk capacitance?
    8. OpAmps & Analog circuit checks
      1. check time delays and slew rates of opamps used as comparators
      2. check opamp input over-drive response for unintended output inversion
      3. check common mode input voltages on opamps and common mode limits for selected supply voltage
      4. connect unused opamps as voltage followers (inverting input to output) with grounded noninverting input and open output to prevent excessive power consumption and uncontrolled oscillations
      5. Unused comparators: All pins to common.
      6. Consider providing additional (unpopulated) components to generalize the opamp circuit later (e.g. convert a voltage follower, to a non-inverting amp)
      7. Check input bias currents of opamps and provide compensation resistors where necessary. (But only there, as they might hurt other performance aspects)
      8. LM324 and LM358 outputs loaded to prevent crossover distortion
      9. amplifiers checked for stability
      10. oscillators checked for reliable startup
    9. Digital circuits
      1. Darlington outputs (1.2v low) driving logic inputs
      2. Reset pins pulled to high/low
      3. Reset is filtered
      4. reset circuit design reliable, both glitch-free and consistent; tested with slow power supply fall time
      5. setup, hold, access times for data and address busses checked (including trace delays)
      6. for buses, ensure bus order matches device order
      7. under-utilization of gates on multi-gate parts checked
      8. symbols identify open collector/drain pins and internal pulled up/down pins
      9. clock lines with series termination and parallel termination component locations present even if not populated; zero ohm resistor for series, unpopulated parts for parallel termination. Select parallel termination voltage rail according to signaling standard (high/low/mid).
      10. avoid direct connect of mode pins or no-connect bus lines to GND or VCC so PCB rework options are maximal
      11. connect DIP switches and other grouped I/O to ports in a logical way, LS bit to LS bit, MS bit to MS bit
      12. Busses
        1. I2C
          1. Pull-up on SDA an SCL I2C is open-drain and thus needs pull-up. References: Texas Instruments, Application Report, I2C Bus Pullup Resistor Calculation
        2. JTAG
          1. Have datasheets been consulted for necessary pull-up/-down? Typically TMS and TDI are pull-up and TCK is pull-down, but it changes between manufacturers and devices. Some devices have internal pull-up/-down, yet external resistors are recommended to keep start-up well-defined.
        3. SWD
          1. Have datasheets been consulted for necessary pull-up/-down? Typically TMS and TDI are pull-up and TCK is pull-down, but it changes between manufacturers and devices. Some devices have internal pull-up/-down, yet external resistors are recommended to keep start-up well-defined.
      13. High-Speed circuits
        1. Matched length connections documented and specified; if possible by schematic capture package, DRC rules have been added.
        2. Differential pair connections documented and specified; if possible by schematic capture package, DRC rules have been added.
    10. RF circuits
      1. RF circuits, components and connections documented and specified; if possible by schematic capture package, DRC rules have been added.
    11. Microcontroller Schematic Checklist
      1. UART/RS-232 RX and TX swap between devices (notations may indicate not what you think: check datasheet)
      2. ability to disable watchdog timer for testing and diagnostics and emulation
      3. use of baud rate friendly clock source for devices that have serial ports
      4. ~IOR and ~IOW strobes on UARTs are typically incompatible with timings of signals readily available on many processors
      5. replacement part compatibility with software requirements: “top-boot vs. bottom boot FLASH”, UART compatibility, SPI memory timing and addressing for different sized parts
      6. all PCB signal changes noted to the software engineers
    12. CPLD / FPGA Schematic Checklist
      1. UART/RS-232 RX and TX swap between devices (notations may indicate not what you think: check datasheet)
    13. EMC / EMI
      1. all unused inputs terminated
      2. unused OpAmps are configured as unity gain followers (if unity gain stable) with grounded input. Shorting both inputs to GND may lead to excessive power consumption.
      3. all outside world I/O lines filtered for RFI
      4. all outside world I/O lines protected against ESD
      5. bypass cap for each IC, close to power pins and low-impedance to GND
      6. consider signal rate-of-rise and fall for noise radiation
      7. separate analog signals from noisy or digital signals
      8. high frequency crystal cases should be flush to the PCB and grounded
    14. Signal integrity (Schematic)
      1. Is there sufficient decoupling?
      2. Is there filtering between analog and digital commons?
      3. Are optocouplers filtered?
      4. Impedance on inputs from outside of board?
      5. Are ferrite beads on input/output power lines?
      6. Are ferrite beads on sensitive signal lines? This includes signal lines that are not sensitive themselves, but connect to sensitive components. E.g. U-Blox recommend ferrite beads on GPS module UART connections to avoid RF noise travelling "along" and into the module, see U-Blox M8 Hardware Integration Manual
      7. Do all ferrite beads have sufficient margin in DC current rating? When the DC current (DC bias current) approaches the maximum, the effective impedance of ferrite beads decreases significantly. See Analog Devices AN-1368, Choosing and Using Ferrite Beads
      8. Is there an estimate of what frequencies the ferrite beads are required to filter?
      9. Do high-speed single-ended digital signals have series resistors?
      10. All high-speed signals have termination resistors/networks if not provided internally by receiving component (terminations may be unpopulated initially)
      11. Is there a footprint for a common mode filter on high-speed differential signals going offboard? For high-speed differential buses like USB 3.0, DisplayPort, and HDMI, a common-mode filter will stop common-mode noise generated in the transceiver from traveling into the signal line where it may radiate from the cable. Notice that the filter might degrade the differential signal slightly.
        Using a footprint allows one to be quickly fitted in case EMC testing fails.
      12. Do op-amps have input filters for EMI? High frequency EMI signals can be rectified by the op-amp if not filtered out beforehand, resulting in DC errors at the output. See Analog Devices MT-096, RFI Rectification Concepts and Analogue circuit design for RF immunity, Clough Consoltants
    15. Testing
      1. Are there ground connection points? (for probes etc.)
      2. Have necessary test points been added?
      3. Have configurable strap-in pins been biased?
      4. Have configurable strap-in pins been connected with jumpers or similar?
    16. Documentation and notes
      1. all components have reference designators and values
      2. Unpopulated parts are clearly marked
      3. Are destinations noted if they go to other sheets
      4. off board connectors identify all signals even if not used on this design
      5. card edge connectors identify mating part
      6. Are connections marked with expected current draw?
      7. Have special PCB or layout requirements been noted? #schematic #pcb
        1. Impedance?
        2. Ground planes?
        3. Routing?
        4. Keep-out?
        5. References to recommendations in datasheet
      8. Notes explaining purpose, functionality, origin, references and caculations for circuits
      9. pin names and attributes on symbols with multi-function pins should match actual design usage (I/O/Bi, Name)
      10. preferred component reference designators used
        1. R fixed resistor
        2. RN resistor network
        3. RV variable resistor
        4. C capacitor (network, fixed or variable)
        5. L inductor
        6. Q transistor, FET, SCR, TRIAC
        7. D,CR diode, rectifier, Zener, varicap, LED
        8. DL multisegment display (any type)
        9. VR,Q,IC voltage regulator
        10. U,IC integrated circuit
        11. J socket, jack (female / receptacle / stationary portion)
        12. P plug (male, disconnecting device / movable portion)
        13. JP jumper (pins, trace, or wire)
        14. Y,X crystal
        15. M modular subassembly, daughter board
        16. S mechanical switch
        17. F fuse
        18. FL filter
        19. T transformer
        20. KB keyboard
        21. B,BT battery
        22. K Relay, magnetically operated switch
    17. Drafting
      1. No overlap between text, notes references, wires, symbols etc. Reference: EEVblog #1129 - Creating a Nice Readable Schematic (7:55)
      2. Is all text horizontal? Within reason and if there is space for it, otherwise vertical, readable from the right side.
      3. Do all components have reference and value?
        1. Are values in a uniform format
        2. Are references using standard designators?
        3. Are references placed unambiguously?
        4. Decimal points avoided? Decimal points might get lost during photocopying. Reference: EEVblog #1129 - Creating a Nice Readable Schematic (44:00)
      4. All junctions dotted
      5. All no-connect pins should be labeled NC (or clearly marked/indicated by other tool mechanism)
      6. No 4-way connections
      7. No upwards pointing ground symbols?
      8. Are component references ordered by schematic layout?
      9. Are the appropriate power nets connected? (Vcc, Vss, Vdd)
      10. Net names on top of lines
      11. Are unused nets left unlabeled? If a net is labeled, it is expected to be used. It is usually clearer to leave it unlabeled.
      12. All connections/markings have a purpose Connections going from and to nowhere lead to confusion.
    18. Sheets
      1. Cover page with general info and table of contents for large schematics
      2. Sheets are consistently sized
      3. Readable when printed
      4. Logical layout should go left-right, top-bottom.
      5. Title block present and consistent on all pages
        1. Name of author
        2. Name of reviewer
        3. List of revisions and changes
        4. Date
        5. Revision
        6. Company / organization
        7. Sheet/drawing number
        8. File name
    19. ERC
      1. ERC and all warnings handled or waived?

  5. Printed circuit board

    1. Technology & Planning
      1. Are gold fingers needed?
      2. How is the PCB panelized?
      3. What stack-up is needed?
      4. Which finish is necessary?
      5. What thickness of finish?
      6. Ability for blind or buried vias?
      7. How is assembly performed?
      8. Board size optimized for panelization (if possible)?

    2. Floorplan

      1. Are mechanical constraints defined?
        1. Mounting
        2. Board size/shape
        3. Connector placement
        4. Human interface placement
      2. Can components be oriented in roughly the same way / consistently in layout?

      3. Are components easily accessible?

        Large components or high density boards might make it difficult to inspect components during testing or after failure. Apart from visual blocking it can also manifest as physically blocking from accessing with probes.
        1. For inspection?
        2. For replacement?
      4. Are interactive components placed in a logical manner? (Consistent orientation of buttons? Consistent rotation of potentiometers?)
      5. Are temperature sensitive components placed away from hot components?
      6. Are EMI sensitive components placed away from noisy parts of the design?
      7. Ground loops made through connections avoided/bypassed
      8. Should there be a ground ring? Ground rings serve both a signal integrity and a practical purpose. For EMC it is necessary to avoid traces along edges of the PCB. Adding a ground ring is an easy way of ensuring this constraint is held. It also provides somewhat better EMI prevention. Source: Why are vias placed this way on a PCB? - Stack Exchange
    3. Footprints / Land patterns
      1. Is pin 1 marked in a consistent manner?
      2. Is component polarity marked in a consistent manner?
        1. For electrically polarized components like capacitors?
        2. For keyed components like connectors?
      3. Are high-density chips marked with pin numbers?
      4. Are there tick-marks for every 5/10 pin on high pin count?
      5. Are there square pins on components? Are the holes big enough?
      6. Have the footprint dimensions been cross-checked with recommended footprint for the specific component?
      7. Are the footprints from the datasheet defined as top view or bottom view?
      8. Are edge-connectors/fingers interleaved/zig-zag? Edge pins on SODIMM are interleaved in placement when comparing the two sides of the inserted board. References:
        • TE connectivity: 390112-1 datasheet
        • Wikipedia: Zig-zag in-line package
      9. Are there the necessary thermal pads?
        1. Are they exposed?
        2. Are they connected to the right net? The net is not necessarily ground.
      10. Are certain pins only accessible on the thermal pad/unexposed pads and is the assembly procedure for this noted? Sometimes the only ground connection is the thermal pad. This requires soldering the thermal pad and it is thus a assembly requirement to ensure it is soldered.
    4. Placement
      1. Are jumpers accessible?
      2. Are debug connectors accessible?
      3. Filter resistors closer to the source?
      4. Termination resistors closer to the target?
      5. Minimum component body spacing
    5. Mechanical
      1. Double check mounting positions will be OK in the equipment the PCB is fitted in.
      2. Is there spacing for an assembly run marking?
      3. Is there clearance for connectors?
      4. Are there mounting holes?
      5. Should mounting holes be electrically isolated?
      6. Should mounting holes have islands?
      7. Should grounded mounting holes have via stitching? Vias around mounting holes improve ground connection and provides some redundancy. Source: Why are vias placed this way on a PCB? - Stack Exchange
      8. Are hole diameters compensating for plating?
      9. Check for clearance problems with screws and mounting hardware.
      10. Mounting holes matched 1:1 with mating parts
      11. Is the outline of the board defined?
      12. Is the mechanical enclosure defined?
      13. Is there enough space for the mating connectors? #clearance #connectors
      14. Is there enough vertical space for components?
      15. Are internal corners rounded? Can they be milled?
    6. Electrical
      1. What stack-up is needed?
      2. Polarized components are oriented correctly
      3. All traces are routed?
      4. Are decoupling capacitors placed close to power pins of ICs?
      5. Are analog and digital commons joined at only one point?
      6. Does ERC pass?
      7. Are isolation barriers large enough? #mechanical #power #safety
      8. Are the appropriate power nets connected? (Vcc, Vss, Vdd)
    7. Signal integrity (PCB)
      1. Digital signals routed over separate (digital) ground planes or within separate board region
      2. Digital and analog signal commons joined at only one point (or use of a continuous, unbroken ground plane with careful component placement)
      3. DGND of ADCs with low digital current draw connected to analog GND plane (Analog Devices: Ask the Applications Engineer – 12: Grounding (Again), Analog Devices MT-031)
      4. High-speed signals avoid gaps in ground planes
      5. Stubs minimized for high-speed signals?
      6. Are there nets that need their length matched/equalised (e.g. fast differential connections such as Ethernet or parallel busses such as DDR lanes)?
      7. Decide whether tightly coupled differencial pairs should be used (based on stackup, manufacturing tolerances, etc.)
        • Tightly coupled diffpairs require very little skew between segments, symmetric and carefully placed vias with GND returns, extra care at component/connector pins due to possibly unavoidable impedance change.
        • Loosely coupled diffpairs require matched flight time for each half of the pair and consistent impedance and progagation delay across the board area. Each half can be designed at (e.g. 50 Ohm) transmission line, maintaining impedance when routing to connectors.
      8. Differential pair spacing based upon impedance calculations with no other vias or signals in between
      9. Transmission lines terminated with an appropriate impedance
      10. Crystal connections are as short as possible
      11. Is there a guard ring around the crystal?
      12. Filters on A/D pins
      13. High speed signals avoid changing reference planes whereever possible. Changing sides of the same reference plane might be a better alternative.
      14. Ground return vias for every layer change of high speed signals
      15. Place I/O drivers and receivers near where their signals leave/enter the board (e.g. at connectors)
      16. Verify that all series terminators are located near the source
      17. Verify that parallel terminators are located near, or even better after the last device on the transmission line (fly by for all devices including last one, ending in termination)
      18. EMI / RFI filters close to entry / exit of shielded areas?
      19. Are traces avoided under sensitive components?
      20. Are traces avoided under noisy components?
      21. Are vias avoided under metal-film resistors?
      22. Is via fencing of sensitive RF transission lines done with the proper via spacing? (< 1/20 lambda)
      23. Is there an option for a shielding can over sensitive circuitry e.g. RF?
      24. Are bypass capacitors close to power pins?
      25. Is low inductance mounting used for decoupling?
      26. Ideally high speed connectors should have the ground plane getting through between pins to avoid signal return paths having to go round the connector to a ground pin on it. GND vias to planes can also help in this regard.
    8. EMC / EMI">
      1. all outside world I/O lines filtered for RFI, observing proper layout
      2. all outside world I/O lines protected against static discharge
    9. RF circuits
      1. Proper RF layout observed and consulted with RF engineer.
    10. Copper pour (Planes / Polygons)
      1. ground planes where possible
      2. Ground / power pins connected and checked
      3. Keep ground planes away under/from high speed OpAmp negative input and feedback, especially when advised to do so by datasheet (may affect stability)
      4. No pour between adjacent pins on ICs? These can be mistaken for shorts during inspection. A keep-out zone between pins can fix this without adjusting the pour clearances.
      5. Planes checked on all layers
      6. Thermal reliefs at appropriate places (e.g. manually soldered components), also for internal power layers
      7. Do planes introduce ground loops?
    11. Traces & Routing
      1. Are trace-pad connections sufficiently obtuse (angle 90 deg or more)?
      2. Tace width/plating and via diameter/plating sufficient for the current carried and max heating
      3. Maximize distances between features where possible
      4. No connections between adjacent pins on ICs? These can be mistaken for shorts during inspection.
      5. Are vias for internal power traces big enough?
      6. Provide multiple vias for high current and/or low impedance traces
      7. No vias under metal-film resistors and similar poorly insulated parts
      8. Is there enough space for heatsinks? #mechanical #thermal
      9. Has mitered bends or soft curves (r > 3 · trace width) been implemented for impedance sensitive traces?
    12. Thermal
      1. Temperature sensitive components (e.g. electolytic caps) placed below / away from hot components
      2. Thermal vias present in thermal pads (no thermal reliefs!)
      3. standoffs on power resistors or other hot components
    13. (Common) PCB Pitfalls
      1. all polarized components checked (capacitors, diodes, transistors, etc.)
      2. USB data lines D+ and D- have 39R series resistors, I2C have pull-up resistors (where not on-chip), Rx and Tx lines are not swapped.
    14. Prototypes
      1. In prototype PCBs without hole plating, traces connect to THT pads from the correct side/layer
    15. Testability & Testing
      1. Testpoints
        1. Are there test points for nets which are difficult to probe due to layout?
        2. PCB has ground turrets
        3. PCB has labeled power rail test points
        4. Test points present for critical circuits, hard to reach (and inner layer) nets, nets with blind or burried or microvias, important signals, important buses and connections to tight SMD chips
        5. test pads for in-circuit or bed-of-nails functional testing
        6. test pad or test via on every net to allow in circuit test
        7. Test pads are (if possible) on only one side of the PCB
        8. Test point on (selected) unused outputs for debug use
        9. Ground connection points close to analog, rf or high-speed test points
        10. Carefully considered the effects of testpoint-stubs on high-speed and rf nets
      2. No logic pins connected directly to power or ground if in-circuit test is planned
      3. In circuit programming connectors are placed near the programmed device
      4. automated netlist check
      5. manual netlist check (at least on some critical examples)
      6. check netlist for nodes with only one connection
    16. Maintenance & Repair
      1. Are there any very big through hole components such as electrolyte capacitors? If so consider not to connect to internal layers on multilayer designs in case they get knocked and the internal connection is damaged. Do use inner layer connections if signal/power integrity demands low impedance.
      2. Layout PCB so that any rework or repair of a component does not require removal of other components
    17. Silk screen
      1. Notes and documentation
        1. Is there a revision number?
        2. Is there a date?
        3. Is there blank space for a serial/assembly number? Remember to consider how large it should be. Reference: Writable area on a PCB - Stack Exchange
        4. Are connector pin-outs labeled?
        5. Fuse size and type marked on PCB
        6. Are functional groups marked?
        7. Are high-density chips marked with pin numbers?
        8. Is functionality labeled?
          1. Test points
          2. LEDs
          3. Buttons
          4. Connectors/terminals
          5. Mounting holes
          6. Jumpers
      2. Drafting
        1. No silk screen on pads
        2. All text is readable from at most two directions
        3. Will the silk screen be legible?
        4. Are component references order by PCB layout?
        5. Is there a coordinate system?
    18. Header/block
      1. Name of author
      2. Name of reviewer
      3. List of revisions and changes
      4. Date
      5. Revision
      6. Company / organization
      7. Sheet/drawing number
    19. Clearance
      1. All keep-out areas honored
      2. Keepout around mounting holes
      3. Clearance for IC extraction tools
      4. Clearance for IC extraction tools
      5. Clearance for programmer/emulator adapter or pod
      6. Clearance for assembly tools (wrenches, screwdrivers etc.)
      7. Clearance for probes
      8. Trace-to-trace clearance based upon voltage rating?
      9. All ICs have pin one clearly marked, visible even when chip is installed
    20. Panelization
      1. Panelized PCB fits test rig
      2. Panel design/layout checked with assembly house (panelized PCB fits test and manufacturing equipment)
    21. Assembly
      1. Is there enough space for the minimum bending radius of the wire harness?
      2. Fiducials (visual references) for automated assembly
      3. Tooling holes for automated assembly
      4. components keep proper distance PCB edge
      5. all polarized components point same way (if possible considering other constraints, e.g. EMC)
      6. Check whether assembly will be performed on pannelized PCBs (if so, check panel layout with assembly company) or separated PCBs (then check if assembly company needs a fixture)
      7. Is there a recommended/necessary order for mounting components on the board?
      8. Will mounting certain components make it impossible to mount others?
      9. Is there an assembly order for subsystems?
      10. Is there a testing order for subsystems?
      11. All legend text reads in one or two directions
      12. Provide ground test points, accessible and sized for scope ground clip
      13. Potentiometers should increase controlled quantity clockwise
      14. Check for traces/pads which may be susceptible to solder bridging (exposed copper)
    22. DFM
      1. Are solder paste openings the proper size?
      2. Clearances respected/checked
      3. Assembly constraints/considerations respected/checked
      4. Are manufacturing tolerances honored?
        1. Solder mask
        2. Silk screen
        3. Traces
        4. Holes
      5. Common tolerance values and design rules followed, if no more specific info is available:
        1. finished hole sizes are ≥10 mils (0.254 mm) larger than lead
        2. pads ≥15 mils (0.381 mm) larger than finished hole sizes
        3. no silkscreen legend text over vias (if vias not soldermasked) or holes
        4. all silkscreen text located to be readable when the board is populated
        5. soldermask does or does not cover vias (ask manufacturer and assembly house if vias are tented)
        6. no acute inside angles in foil
        7. components ≥200 mils (5.08 mm) from edge of PCB
        8. traces ≥ 20 mils (0.508mm) from edge of PCB
        9. silkscreen legend text weight ≥10 mils (0.254 mm)
    23. Manufacturing documents & documentation
      1. Are all manufacturing requirements noted on the layout file (or in additional document)?
        1. PCB stackup noted:
          1. layer count,
          2. holes and via types and spans,
          3. for dielectrics: material type, weave, thickness, prepreg or core,
          4. for copper: material, copper weight/thickness, surface roughness
          5. soldermask, surface finish,
          6. etc.
        2. Layers/Traces with controlled impedance specified (if applicable). This is normally related to the stack-up and should be discussed during stackup negotiation.
        3. Hole diameter on drawing are finished sizes, after plating.
        4. Thru hole drill tolerance noted
        5. Plugged (and plated over) vias are documented.
        6. Blind vias / Microvias are documented.
        7. drill legend shows all symbols and sizes
        8. soldermask over bare copper noted if needed
        9. Who determines soldermask expansion (customer or sometimes manufacturer),
        10. Is copper balancing allowed by the manufacturer (on which layers/in which regions),
        11. How many individual boards are on a PCB (if it has been panelized),
        12. How boards are going to be cut (e.g. v-scoring or routing, if panelized),
      2. Parts Lists (BOM)
        1. each component has quantity, reference designator and description (and manufacturer and part number / order number)
        2. suggested and alternate manufacturer(s) listed
        3. object/binary code and method/programmer specified for each programmable device
      3. drill origin is a tooling hole
      4. NC drill and photoplot file language format noted
      5. tools on drill plot and NC drill file cross checked
      6. Check output files:
        1. manufacturing output files checked in file viewer
        2. All layers are aligned
        3. All layers are present (file exists)
        4. All layers have data (not just vias)
        5. Is there a drill legend?
        6. Board outlines/layers have the correct dimensions
        7. Fill polygons have the right isolation, orphan settings and minimum neck width
        8. Make sure your soldermask is correct near high-density parts (tented vias, etc.)
        9. No tool errors spotted (output not matching design files)
        10. Verify a netlist generated from output files against design netlist (if you have the tools to do so)
        11. Test coupon on PCB containing minimum geometry features
        12. Layer counter at PCB edge

      1. Does ERC pass?
      2. Does DRC pass?
      3. Are there any superfluous vias?
      4. Does LVS pass? Layout Versus Schematic is a check to confirm that the board layout actually implements the schematic. That is: Are the traces actually connecting the proper components? Resources: Wikipedia.
